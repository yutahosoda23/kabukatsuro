# おはよう＝今朝の市場まとめ＋朝の株候補メール送信

「おはよう」と言われたら、このルーティンを実行して当日の朝メールを送る。
ローカルの自動実行版は `scripts/run_morning.sh`（launchd・毎営業日8:00）。
このスキルは **市場まとめ（WebSearchが必要な部分）も付けて手動で送る** 拡張版。

作業ディレクトリは `/Users/yuthsd/kabukatsuro`。
プロジェクトの `.venv` と `.env.local` をそのまま使う（認証情報は `.env.local` にあり、リポジトリにはコミットしない）。

メールの構成（この順序）:
1. 🌅 **今朝の市場のまとめ**（地合い・主要指数・注目イベントを手早く把握）
2. 🏇 **デイトレ適性ランキング（15,000円以下）** TOP10（寄付前・値動き×流動性／2,300円以下は除外／`matsui.py`でスクレイピング）
3. 🏇 **デイトレ適性ランキング（2,300円以下）** TOP10（少額版）
4. 🔥 **お祭り銘柄 TOP10**（デイトレ予習）＋ 🔄 急落リバウンド候補
5. 📈 日足GC直前ランキング **TOP5**（5日線/25日線・最下部）
6. 💡 GC直前（株価2300円超）参考リスト **TOP5**（最下部）

## 手順

### 0. 準備（環境変数の読み込み・依存確認）
```bash
cd /Users/yuthsd/kabukatsuro
set -a; . ./.env.local; set +a          # GMAIL_USER / GMAIL_APP_PASSWORD / TO_EMAIL を読み込む
.venv/bin/python -c "import yfinance,pandas,xlrd,jpholiday,requests" 2>/dev/null \
  || .venv/bin/pip install -q yfinance pandas xlrd jpholiday requests
```

### 0.5 ★「おはよう」のたびに毎回フル送信する（返信モードは廃止）
**launchdの自動送信が既に届いていても、毎回フル（市場まとめ＋全ランキング）で新規送信する。**
以前あった「送信済みなら市場まとめだけ返信する」モードは廃止し、**同じ内容のメールを毎回送る**。
朝メールはMessage-IDを固定しないので、同日に複数回送ってもGmailで重複扱いされず新着として届く。
（`--check-sent` での分岐は不要。休場日判定は次のスクリーニングが行い、SKIPなら中止する。）

## フル送信（市場まとめ＋全ランキングを新規送信）

### 1. テクニカルスクリーニング（全銘柄。1〜3分かかる）
```bash
.venv/bin/python scripts/screener.py --out /tmp/kabu_screen.json
```
- 出力に `SKIP`（終了コード10）が出たら本日は休場日（土日・祝日）。
  **以降を中止し、メールは送らず「休場日のためスキップ」と報告して終了する。**

### 2. お祭り銘柄（デイトレ予習）を生成
```bash
.venv/bin/python scripts/daytrade.py --out /tmp/kabu_matsuri.json
```
Yahoo値上がり率・出来高ランキングのスクレイピング＋日足テクニカル＋履歴分析で
「お祭り銘柄 TOP10」と「急落リバウンド候補」を自動生成する（失敗しても空で続行）。

### 2.5 松井証券デイトレ適正ランキングを取得
```bash
.venv/bin/python scripts/matsui.py --out /tmp/kabu_matsui.json
```
松井証券の寄付前「デイトレ適正ランキング」（値動き＝株価変動率×流動性）を
スクレイピングして上位15件を取得する（失敗しても空で続行）。

### 3. 今朝の市場のまとめを作る → /tmp/kabu_market.json ★このスキルの肝（A・B共通）
WebSearch で**当日朝の地合い**を調べ、デイトレーダーが寄り前に手早く要点をつかめるJSONを作る。
ポイントは「有益な情報を手早くキャッチできること」。盛り込みすぎず要点に絞る。

調べる内容:
- 主要指数の前日終値（日経平均・TOPIX・グロース250）と前日比
- 米国市場のオーバーナイト（ダウ・S&P500・ナスダック・SOX半導体）の終値/方向
- ドル円（円安/円高の向き）、必要なら日経先物(CME/夜間)・原油
- 今日の注目イベント（経済指標・決算・政策）と物色テーマ（AI半導体・防衛 等）
- 一言の地合いコメント（リスクオン/オフ、高値警戒など）

**データの鮮度に注意**：米国終値など当日朝に未確定なものは「暫定」「確報待ち」と明記する。
事実が取れない数値は推測で埋めず省く。

保存形式（`dir` は up=赤▲ / down=青▼ / flat=灰）:
```json
{
  "as_of": "2026/06/03 朝・寄り前",
  "sentiment": "一言で地合い。リスクオン/オフ、注目テーマ、警戒点。",
  "stats": [
    {"label": "日経平均（前日終値）", "value": "66,734", "chg": "-0.30%", "dir": "down"},
    {"label": "ドル円", "value": "159.7円台", "chg": "円安", "dir": "flat"}
  ],
  "watch": ["テーマ: …", "注意: …", "マクロ: …"]
}
```

### 4. メール送信
```bash
.venv/bin/python scripts/mailer.py morning \
  --screen /tmp/kabu_screen.json \
  --matsuri /tmp/kabu_matsuri.json \
  --market /tmp/kabu_market.json \
  --matsui /tmp/kabu_matsui.json
```
送信前に `build_morning` でドライレンダリングしてセクション順（市場まとめ→GC→お祭り→松井）を確認すると安全。

### 5. 報告
市場まとめの要点、各セクションの銘柄数、送信先、エラーの有無を簡潔に報告する。

## お祭り銘柄の見つけ方（本で学んだ手法 — `daytrade.py` が自動化済み）

デイトレで狙うのは「強い上昇エネルギーを持つお祭り銘柄」。`daytrade.py` が以下を自動でスコア化する。
WebSearchで材料を補足したいときの観点としても使う。
- 値上がり率上位 ∪ 出来高上位を母集団に、**25日移動平均線の当日上抜け**・**出来高急増**を重視
- 材料（決算/分割/提携等）を kabutan適時開示と照合。**材料なしの急騰＋出来高急増は「仕手」っぽい動き**としてフラグ
- **IPOホヤホヤ**、**ランキング常連**、**連動銘柄**、**急落リバウンド候補**を履歴から判定

### 補足ソースの注意
- **夜間PTSの「買付未成率・件数」** は需給の偏りを示す強いシグナルだが、**無料の公開ソースが無い**
  （証券会社のログインが必要）。代替として `catalysts.py` は**掲示板投稿数（注目度）**を利用する。
- WebSearchで材料銘柄を補足するときは、決算（上方修正・増配）/ 目標株価引き上げ / 自社株買い / M&A・新製品 を探し、
  **必ずニュースで裏取り**してから採用する。
